Trade Order Tracker

(aka a dumping ground to practice Spring Boot, Docker, Microservices, AWS, ActiveMQ, REST API ect)

This is a REST application. The purpose is take in and give out data. There is no UI component other than this basic home page. It would be best to use a tool like Postman to view the JSON objects returned from the endpoints.


HTTP METHOD URL Description
Users
GET /portfolio/user/ All Users with all related data
GET /portfolio/user/{id} Show a specific User with all Details
POST /portfolio/user/

Add a User:
Body Example:
{ "employeeId": "1023322", "firstName": "Julie",
"lastName": "Benice", "address1": "8 Acorn Ave",
"address2": null, "city": "Bedford",
"state": "NH", "zipCode": "03032",
"phoneNumber": "603-215-3434",
"email": "julie.benice@gmail.com" }
Groups
GET /portfolio/group/ Get Groups with all related data
GET /portfolio/group/{id} Get a specific Group with all related data
POST /portfolio/group/

Add a Group
Body Example:
{ "groupName" :"Brick Stone Bros",
"role" : "USER" }
Trade Order
A Trade Order is a purchase of a number of shares of stock
GET /portfolio/tradeorder/ Get All Trade Orders
GET /portfolio/tradeorder/{id} Get a specific Trade Order (it also pulls the latest price)
GET /portfolio/tradeorder/user/{id} Get all Trade Orders for a specific user
POST /portfolio/tradeorder/

Add a Trade Order
Body Example:
{
"side": "BUY", //BUY or SELL
"security": "RTN",
"fund": "MAG",
"quantity": 10000,
"employeeId": "1020552",
"groupId": 66666
}
NOTE:"Security" Needs to be a valid security / symbol. See /portfolio/stock/ about finding valid securities
NOTE:"employeeId" and "groupId" need to be valid groups.

Data
GET /portfolio/reports/summary/user/update/ Updates the Portfolio Summary. (This sends a message to an ActiveMQ broker -hosted on AWS)
GET /portfolio/reports/summary/user/ Get Portfolio Summaries for each User
GET /portfolio/reports/summary/user/{id} Get Portfolio Summary for a specific User
GET /portfolio/reports/summary/security/user/{employeeid} Get Summaries for all the Securities for a specific User
GET /portfolio/reports/summary/security/{symbol}/user/{employeeid} Get Summaries for all the Securities for a specific User and Security using the Symbol
GET /portfolio/reports/summary/fund/user/{employeeid} Get Summaries for all the Funds for a specific User
GET /portfolio/reports/summary/fund/{fund}/user/{employeeid} Get Summaries for all the Funds for a specific User and Fund
History
GET /portfolio/history/ Get Daily Value of the Portfolio for all users. This gets captured once a day.
GET /portfolio/history/user/{id} Get Daily Value of the Portfolio for a specific User. This gets captured once a day.
Stock Information
This is pulled from a microservice that sole purpose is to provide Stock information to update the portfolio value
GET /portfolio/stock/{symbol} Pulls the latest information about the Stock quote "Symbol" (ie "T" for AT&T)
GET /portfolio/stock/ Pulls a list of all available Stock "Symbols"
GET /portfolio/updateHistory/ Utility Link - updates the Daily History for all users. Only use if Server gets reset.
